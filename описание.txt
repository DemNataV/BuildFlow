1.  ЦЕЛЬ ПРОГРАММЫ
--------------------------------------------------------------------------------
Создать интеллектуального ассистента, который помогает частным заказчикам 
(физическим лицам) и небольшим строительным компаниям:

- Структурировать хаотичные пожелания заказчика в формальный план работ.
- Автоматически выявлять скрытые зависимости между этапами строительства.
- Предотвращать ошибки планирования (например, "заказать бетон" до того, 
  как вырыт котлован).
- Генерировать прозрачную смету с разбивкой по дням, этапам и микро-операциям.
- Создавать визуальный граф (схему) строительства для контроля подрядчиков.


2.  КЛЮЧЕВАЯ ИДЕЯ (НОУ-ХАУ)
--------------------------------------------------------------------------------
Программа строит не просто список работ, а **ДИНАМИЧЕСКИЙ ГРАФ СОСТОЯНИЙ**.
Вместо привычного "сделать А, потом Б, потом В", мы оперируем понятиями:

- **Состояние (State)** — факт завершенности этапа (например, "Котлован_вырыт").
- **Действие (Action)** — работа, которая переводит из одного состояния в другое.
- **Зависимость** — Действие нельзя начать, пока не достигнуты все init_states.

Такой подход позволяет:
- Автоматически перестраивать план, если заказчик меняет решение 
  (например, добавляет подвал).
- Находить узкие места и "мертвые зоны" (например, 7 дней простоя бетона, 
  которые можно занять бурением скважины).
- Предлагать оптимальные параллельные маршруты.

Программа состоит из 3-х модулей: 
  1) Опросник (сбор данных с ветвлениями).
  2) Генератор плана (декомпозиция + правила логики).
  3) Экспортер (JSON для фронтенда + печатная смета).


3.  МОДУЛЬ 1: ИНТЕЛЛЕКТУАЛЬНЫЙ ОПРОСНИК
--------------------------------------------------------------------------------
Опросник адаптивный: следующие вопросы зависят от предыдущих ответов.
Цель — собрать ровно столько данных, сколько нужно для конкретного проекта,
не перегружая пользователя.

3.1. БЛОК "БАЗА" (Обязателен всегда):
      - Тип объекта (Дом/Баня/Гараж/Хозблок)
      - Общая площадь (м²)
      - Этажность
      - Количество спален и санузлов

3.2. БЛОК "УЧАСТОК" (Если есть земля):
      - Регион (влияет на коэффициент цен и сроков)
      - Наличие перепадов высот (включает "Вертикальную планировку")

3.3. БЛОК "ГЕОЛОГИЯ" (Зависит от "Проводили ли геологию?"):
      - Тип грунта (Скала/Песок/Глина/Торф) -> автоматический выбор типа 
        фундамента (Ленточный/Свайный/Плитный)
      - Если геологии нет -> предлагаем "Универсальный свайный" и добавляем 
        этап "Геологические изыскания" в план.

3.4. БЛОК "КОММУНИКАЦИИ" (Логическое дерево):
      - Вода: Есть центральная? 
          -> НЕТ: добавляем "Бурение скважины" (спрашиваем глубину).
          -> ДА: добавляем "Врезка в водопровод".
      - Канализация: Есть центральная?
          -> НЕТ: добавляем "Монтаж септика".
          -> ДА: добавляем "Врезка в канализацию".
      - Газ: Есть магистраль?
          -> ДА: добавляем "Подведение газа".
          -> НЕТ/Не знаю: пропускаем (используем электричество).
      - Электричество: Какая мощность?
          -> Менее 5 кВт: добавляем "Увеличение мощности" в план.

3.5. БЛОК "КОНСТРУКТИВ" (Выбор технологий):
      - Тип стен (Кирпич/Газобетон/Каркас) -> влияет на время кладки.
      - Наличие подвала -> добавляет "Дренаж" и "Гидроизоляцию".
      - Тип кровли (Скатная/Мансардная) -> для мансарды автоматически 
        добавляем "Утепление кровли".
      - Отделка: Черновая или Под ключ? 
          -> "Под ключ" добавляет внутренние работы (электрика, сантехника, 
             плитка).


4.  МОДУЛЬ 2: ГЕНЕРАТОР ДЕТАЛЬНОГО ПЛАНА (БАЗА ЗНАНИЙ)
--------------------------------------------------------------------------------
Программа содержит встроенную "Библиотеку микро-действий". 
Каждое верхнеуровневое действие (Фундамент, Стены, Кровля) разбивается на 
атомарные шаги с жесткой логикой переходов.

4.1. ПРИМЕР ДЕКОМПОЗИЦИИ "ЗАЛИТЬ ФУНДАМЕНТ":
      Шаг 1: Очистка участка       [init: Участок_куплен] 
                                   -> [final: Участок_очищен]
      Шаг 2: Разметка              [init: Участок_очищен] 
                                   -> [final: Разметка_готова]
      Шаг 3: Котлован              [init: Разметка_готова] 
                                   -> [final: Котлован_вырыт]
      Шаг 4: Подушка (песок/щебень)[init: Котлован_вырыт] 
                                   -> [final: Подушка_готова]
      Шаг 5: Опалубка              [init: Подушка_готова] 
                                   -> [final: Опалубка_установлена]
      Шаг 6: Арматура              [init: Опалубка_установлена] 
                                   -> [final: Каркас_связан]
      Шаг 7: Заливка бетона        [init: Каркас_связан] 
                                   -> [final: Бетон_залит]
      Шаг 8: Уход (7 дней)         [init: Бетон_залит] 
                                   -> [final: Бетон_набрал_прочность]
      Шаг 9: Демонтаж опалубки     [init: Бетон_набрал_прочность] 
                                   -> [final: Опалубка_снята]
      Шаг10: Гидроизоляция         [init: Опалубка_снята] 
                                   -> [final: Фундамент_готов]

4.2. КЛЮЧЕВЫЕ ОСОБЕННОСТИ БАЗЫ ЗНАНИЙ:
      - Каждому микро-действию присвоены:
          * Стандартное время в днях (среднее по рынку).
          * Базовая стоимость в рублях.
      - Коэффициенты: Время и стоимость умножаются на коэффициент региона 
        (Москва = 1.4, Краснодар = 1.0, Новосибирск = 1.1).
      - Умное исключение: Если дом одноэтажный, этап "Кладка второго этажа" 
        автоматически удаляется. Если кровля не мансардная, "Утепление" 
        удаляется.

4.3. ДОПОЛНИТЕЛЬНЫЕ СИСТЕМНЫЕ ДЕЙСТВИЯ (Добавляются автоматически):
      - **Бытовка для рабочих** (добавляется всегда, если есть участок).
      - **Склад для материалов** (добавляется всегда).
      - **Геология** (если не проводилась ранее).
      - **Вывоз строительного мусора** (назначается после каждого этапа).


5.  МОДУЛЬ 3: ФОРМАТ ВЫХОДНЫХ ДАННЫХ (JSON)
--------------------------------------------------------------------------------
Программа генерирует единый JSON-файл, состоящий из трех разделов:

5.1. РАЗДЕЛ "USER_ANSWERS" (сырые ответы):
      Хранит историю опроса для аудита и повторного использования.

5.2. РАЗДЕЛ "GRAPH_NODES" (для отрисовки):
      Плоский список всех микро-действий. Каждый узел содержит:
      {
        "action": "Земляные работы",
        "init_states": ["Разметка_готова"],
        "final_states": ["Котлован_вырыт"],
        "time_days": 3,
        "cost_rub": 90000,
        "phase": "Фундамент"
      }
      ЭТОТ ФОРМАТ ИДЕАЛЬНО ПОДХОДИТ ДЛЯ БИБЛИОТЕК ВИЗУАЛИЗАЦИИ ГРАФОВ 
      (vis.js, D3.js, Mermaid, PlantUML).

5.3. РАЗДЕЛ "DETAILED_PLAN" (иерархический план для заказчика):
      Данные сгруппированы по этапам (Фундамент, Стены, Кровля...). 
      Для каждого этапа подсчитаны общие сроки и бюджет.
      Это готовый документ для распечатки и передачи прорабу.


6.  ВИЗУАЛЬНОЕ ПРЕДСТАВЛЕНИЕ (ИДЕЯ ДЛЯ UI)
--------------------------------------------------------------------------------
Программа отрисовывает граф, где:

- **Квадраты (или круги)** — это Состояния (например, "Котлован_вырыт").
- **Прямоугольники с закруглениями** — Действия (например, "Залить бетон").
- **Стрелки** — показывают поток работ.

Цветовая кодировка:
  - Зеленый: Выполнено.
  - Желтый: В процессе (сегодня).
  - Серый: Еще не начато.
  - Красный: Заблокировано (нет ресурсов/материалов).

При клике на действие открывается всплывающее окно с:
  - Сметой по данному этапу.
  - Списком необходимых материалов.
  - Технологической картой (ссылка на видео или инструкцию).


7.  РАСЧЕТ ВРЕМЕНИ И ДЕНЕГ (АЛГОРИТМ)
--------------------------------------------------------------------------------
7.1. ВРЕМЯ:
      - Базовое время микро-действия суммируется последовательно.
      - Если действия независимы (не имеют общих init_states), программа 
        предлагает выполнять их параллельно. (В будущем будет реализован 
        расчет критического пути по методу PERT/CPM).

7.2. ДЕНЬГИ:
      Формула: 
      Итоговая_цена = (Базовая_цена * Региональный_коэффициент) 
                      + (Базовая_цена * Коэффициент_сложности_грунта) 
                      + (Базовая_цена * Коэффициент_этажности) 
                      + Логистическая_надбавка (если регион удаленный).

7.3. ПРИМЕР РАСЧЕТА:
      Заливка фундамента в Москве (коэф. 1.4) на глине (коэф. 1.3) 
      2-этажного дома:
      База = 450 000 руб.
      Итого = 450 000 * 1.4 * 1.3 * 1.1 (за этажи) = 900 900 руб.

9.  ТЕХНИЧЕСКИЙ СТЕК (РЕКОМЕНДАЦИИ)
--------------------------------------------------------------------------------
- Бэкенд: Python 3.10+ (легко писать бизнес-логику и парсить JSON).
- Хранение данных: JSON файлы (для малых проектов) или SQLite (для истории).
- Фронтенд для отрисовки графа: JavaScript + библиотека vis.js или 
  React-обертка над Mermaid.
- Мобильная версия: Progressive Web App (PWA).


10. ЗАКЛЮЧЕНИЕ
--------------------------------------------------------------------------------
Программа решает проблему "информационного хаоса" в частном строительстве. 
Она превращает разрозненные пожелания в инженерно обоснованный, 
проверенный на логические ошибки план. 

Прозрачность сметы и визуализация графа позволяют заказчику чувствовать 
себя уверенно при общении с подрядчиками, а строителям — четко понимать 
последовательность задач и избегать простоев.
